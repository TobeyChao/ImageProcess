# 色键抠图（Chroma Key）

通过泛洪填充+容差+边缘柔化算法移除纯色背景，效果等同 Photoshop 魔术橡皮擦。

## 输入解析

从用户消息中提取以下参数：

| 参数 | 标志 | 必填 | 说明 |
|------|------|------|------|
| 输入图片 | `-i` | ✓ | 待处理的图片路径 |
| 背景色 | `-c` | 否 | Hex 色值（如 `#FFFFFF`）或 `auto` 自动检测，默认 `auto` |
| 容差 | `-t` | 否 | 颜色距离阈值 1-255，默认 32 |
| 输出路径 | `-o` | 否 | 默认：输入文件名 + `_chroma.png` |

若用户未提供输入图片路径，用 **AskUserQuestion** 工具询问。

## 执行步骤

### 1. 运行脚本

从项目根目录执行：

```bash
python .claude/skills/chroma-key/scripts/chroma_key.py -i "<输入图片>" [-c "<背景色>"] [-t <容差>] [-o "<输出路径>"]
```

- 背景色支持 Hex 格式（`#RRGGBB` 或 `#RGB`）或 `auto` 自动检测（从四角采样）。
- 容差越大，抠除的颜色范围越宽。32 适合大多数纯色背景，可调至 10-64。
- 脚本输出 `Saved: <路径>` 表示成功。

### 2. 报告结果

**成功**：告知输出文件路径，可附上简要说明（如文件大小、原始分辨率）。

**失败**：展示错误信息，并按以下思路排查：

| 错误特征 | 可能原因 | 建议 |
|----------|----------|------|
| 颜色格式错误 | Hex 格式不正确 | 使用 `#RRGGBB` 格式，如 `#FF0000` |
| `ModuleNotFoundError` | 依赖未安装 | `pip install pillow numpy` |
| 输入文件不存在 | 路径错误 | 请用户确认路径 |

依赖安装命令：
```bash
pip install pillow numpy
```

---

**注意**：输出固定为 RGBA PNG；算法通过从四边泛洪填充确保主体上同色区域不被误删；适合纯色/近纯色背景，复杂背景请用 rmbg。
