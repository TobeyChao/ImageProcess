# 色键抠图算法文档

## 概述

针对纯色背景 + 不透明主体的图片，通过泛洪填充定位连通背景区域，再按颜色距离映射 alpha 通道，输出 RGBA 透明 PNG。效果等同 Photoshop 魔术橡皮擦（连续模式）。

```
输入: RGB 图片 + 目标背景色 C + 容差 T
输出: RGBA 图片（背景透明，主体保留）
```

## 算法步骤

### 第一步：颜色距离计算

对每个像素 p，计算其与目标背景色 C 的欧几里得距离：

```
d(p) = sqrt[(R_p - R_C)² + (G_p - G_C)² + (B_p - B_C)²]
```

值域：`[0, ~441.67]`（纯黑到纯白的距离）。

### 第二步：候选背景 mask

```
candidate(p) = d(p) ≤ T
```

距离在容差范围内的像素被标记为"背景候选"。容差 T 是 RGB 空间中的球半径。

### 第三步：四边泛洪填充（Flood Fill）

这是算法的核心——只移除**连通到图片边缘**的背景区域，避免误删主体上与背景同色的孤立区域。

```
1. 初始化空的 visited 集合和队列
2. 遍历图片四边（y=0, y=H-1, x=0, x=W-1）：
   若 candidate(p) = true，将 p 加入队列并标记 visited
3. BFS（4 连通）：
   while 队列非空：
     取出像素 q
     对 q 的上下左右邻居 n：
       若 candidate(n) = true 且 n 未被访问：
         标记 visited，加入队列
```

4 连通（仅上下左右）而非 8 连通，避免从角落"渗透"到主体内部。

**为什么必须连通到边缘？** 考虑一个蓝色按钮放在蓝色背景上：按钮上的蓝色像素和背景蓝色像素颜色完全相同，但它们不连通——中间隔着按钮的边框。泛洪填充只会删掉从边缘可达的背景区域，按钮上的蓝色像素不受影响。

```
┌──────────────────────┐
│ 背景 #4A90D9         │  ← 连接到上边缘 → 被标记
│  ┌──────────────┐    │
│  │ 按钮 #4A90D9  │    │  ← 颜色相同但不连通 → 保留
│  │ (同色！)       │    │
│  └──────────────┘    │
│ 背景 #4A90D9         │  ← 连接到左/右/下边缘 → 被标记
└──────────────────────┘
```

### 第四步：Alpha 通道映射

对泛洪填充区域内的像素，将颜色距离线性映射为 alpha：

```
t = d(p) / T                (clamped to [0, 1])
alpha(p) = t × 255
```

| 距离 | t | alpha | 效果 |
|------|---|-------|------|
| d = 0（完全匹配） | 0 | 0 | 完全透明 |
| d = T/2 | 0.5 | 128 | 半透明 |
| d = T（边界）| 1 | 255 | 不透明 |

线性映射使边缘平滑过渡——抗锯齿像素（背景色和前景色的混合）获得中间 alpha 值，在合成到新背景时不会有生硬的锯齿边缘。

对于不在泛洪区域内的像素（即主体内部），alpha 恒为 255（完全不透明）。

### 第五步：合成输出

将原始 RGB 与 alpha 通道合并为 RGBA 四通道图像。

```
result = concat(RGB, alpha)  →  (H, W, 4) RGBA PIL Image
```

## 背景色获取方式

三种方式殊途同归，最终都得到一个 (R, G, B) 元组，传入同一算法：

| 方式 | 实现 | 使用场景 |
|------|------|----------|
| 自动检测 | 从图片四角各取 5×5 像素，合并取中位数 | 默认，背景均匀分布 |
| 点击取色 | `gr.Image.select` 获取点击坐标，取该像素 RGB | 背景非纯色但有点击位置 |
| 手动输入 | 解析 `#RRGGBB` 或 `#RGB` Hex 字符串 | AI 生图已知背景色 |

### 自动检测细节

```
1. 四角坐标：左上(0,0)、右上(W-5,0)、左下(0,H-5)、右下(W-5,H-5)
2. 每个角取 5×5 区域 → 共 100 个像素（图片 ≥ 6×6 时）
3. 对每个通道取中位数（比均值更抗噪）
4. 返回 median_R, median_G, median_B
```

若图片 < 6px，crop 区域自动 clamp 到图片边界内，取尽可能多的像素。

## 容差选择指南

| 容差 | 适用场景 |
|------|----------|
| 1-10 | 纯色背景（如 AI 生成指定色值的纯色底） |
| 16-32 | 近纯色背景，有轻微压缩噪声（推荐默认 32） |
| 32-64 | 有轻微渐变或 JPEG 压缩伪影的背景 |
| 64-128 | 背景有明显色差但仍可算"单一色系" |

容差是 RGB 空间中的半径。例如 #FFFFFF 到 #F0F0F0 的距离 ≈ 21，容差 32 可覆盖。

## 边界情况处理

| 情况 | 处理 |
|------|------|
| 容差 = 0 | 只看精确匹配的像素，不做除法（避免 NaN） |
| 容差 ≥ 255 | 几乎所有像素都是候选，可能过度抠除 |
| 图片 < 6×6 | auto_detect 采样区域自动 clamp 到边界内 |
| 主体触及图片边缘 | 该区域也可能被泛洪标记，需要手动调整容差或改用点击取色指定种子点 |
| 背景被主体分割成多块 | 只要每块都连通到任意一条边，都会被正确标记 |

## 复杂度

- 时间：O(H × W)，每个像素被访问常数次
- 空间：O(H × W)，存储距离矩阵、visited 矩阵和 BFS 队列
- 无 GPU 依赖，纯 numpy 实现，1000×1000 图片处理时间 < 100ms

| | rmbg (BiRefNet) | bwdiff (黑白差分) | chroma-key (色键) |
|---|---|---|---|
| 算法 | 深度学习 | 像素差值反算 | 泛洪填充 |
| 输入 | 1 张图 | 2 张图（黑底+白底） | 1 张图 + 颜色 |
| 适用 | 复杂自然背景 | 任何背景（需拍两张） | 纯色背景 |
| GPU | 推荐 | 不需要 | 不需要 |
| 边缘质量 | 时有模糊 | 精确（数学反算） | 锐利（线性映射） |
